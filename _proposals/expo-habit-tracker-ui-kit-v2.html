Pulse — a habit-tracking pocket instrument.

The selected direction. The build started as two candidates; the machined-hardware instrument won and is renamed Pulse, and the light athletic console it beat is folded back in as a day-mode theme. One app, two themes, a coach you can dial.

✓ Built progression — shipped 2026-05-23

v0–v6 of demos/expo-habit-tracker-v* are live at vschool.github.io/AAM-Demos/expo-habit-tracker-v{0..6}/. The Habit detail mockup below shipped at v6 (dynamic routes — app/habit/[id].tsx). The Add-habit dedicated screen and the Signature moment screen are intentionally deferred — the inline AddChannelPanel covers add (v2), and the swipe-to-complete animation lives in-row (v3 ThrowRow) rather than as a separate screen.

✓ Locked build spec

  1. Name: the app is Pulse across v0–v5 (the "Knack" hardware language is kept as the look; the name is retired into the brand).
  2. Dark theme (default): the machined instrument below — black + aluminum + indicator LEDs + amber segment readouts. This is the hero look.
  3. Light theme (day mode): the same instrument below, re-skinned — silver face, light panels, positive LCD, same LEDs + reserved roles. A real in-app theme toggle (☾/☀) flips dark↔light (it also serves as the guideline's PaletteToggle chrome).
  4. Net-new feature — Coach tone dial: a hardware-style mode selector (Chill / Firm / Elite) that sets reminder personality; rides the v5 push-notification beat.
  5. Reserved roles stay constant across both themes — done / today / streak / skipped-rest each keep one meaning; only the hex changes per theme (LED palette in dark, lime/jade/coral/slate in light).
  6. Approver: Emily. Next: scaffold v0 in a fresh session (no code this session).

How to read the two themes below

  • The two sections are now the two themes of one app, not competing directions. § Dark theme is the default machined-instrument look; § Light theme is day mode. The theme toggle switches between them.
  • The status row is the most diagnostic artifact. Watch the four locked states — done / today / streak / skipped-rest. Both themes use the same indicators: physical indicator LEDs + a segment streak counter. Dark glows them on black; Light shows the same hues on a silver face with a positive (dark-digit) LCD. Roles never change; neither theme punishes a miss.
  • Each theme shows six real phone screens in device frames — Today, Add-habit, Habit detail, Stats/momentum, Settings, and the signature moment — covering the brief's full IA.
  • Swatches marked ★ are reserved status colors: one semantic role each, never decorative (guideline §7).
  • The hero motion moment ("The Throw") maps to a Reanimated v3 primitive, gated behind useReducedMotion(). Full v0→v5 progression in Motion.
  • Palettes are sourced — see Sources (live vs. inferred vs. the seed design Emily provided).
Pulse · Dark theme · default

The instrument

the machined-hardware look (was "Knack") · "a habit is a switch you throw" · + Coach tone dial (Chill/Firm/Elite, v5)

A habit tracker as a pocket instrument — a piece of dedicated hardware, not an app pretending to be paper. Machined aluminum, screen-printed labels, and a row of indicator LEDs. Each habit is a channel with a physical toggle; your streak is an amber segment readout that glows like a real device. Completion isn't a checkbox going gray — it's throwing a switch with a satisfying mechanical snap. Restraint and function over decoration; the device tells you its whole state at a glance from across the room.

Anchored on Teenage Engineering (pure-black canvas, brushed-aluminum panels, screen-print labels, one orange accent · live-inspected) + Playdate / Panic (playful hardware confidence, segment displays, chunky geometric type · live-inspected).

⏻

Hero motion — "The Throw" (lands at v3, the swipe-gesture beat)

Swiping a channel drives the toggle knob across its track with an overshoot, then it settles — the LED color-snaps amber→green at the moment of contact and a sharp haptic fires. Built on Gesture.Pan() feeding a useSharedValue, released into withSpring({ damping: 12, stiffness: 220 }) for the mechanical snap, with Haptics.impactAsync(Heavy) on threshold. Gated by useReducedMotion() → instant state flip, no travel.

Pulse · Light theme · day mode

Day mode

the SAME instrument, lit for daytime · flip it with the in-app theme toggle (☾ ⇄ ☀)

This is not a second app — it's the same Pulse instrument in light mode. Identical screens, channels, switches, indicator LEDs, segment readouts, meter-bridge, and the Coach dial; the only thing that changes is the face. The canvas goes from black to brushed silver, the panels lighten, and the glowing LCDs invert to a positive readout (dark digits on a pale LCD, like a Casio in daylight). The four reserved roles keep their exact meanings — the green LED is still "done," the amber still "now." The theme toggle is a single switch; everything else is theme-agnostic.

Anchored on Playdate (light-grey hardware body, black labels, daylight-readable face · live-inspected) + Teenage Engineering (brushed-silver aluminum faces · live-inspected). The "go light" idea was seeded from an earlier light athletic exploration; here it's applied to the instrument rather than shipped as a separate design.

☀

Motion is theme-agnostic — same hero as dark

The light theme runs the exact same motion as dark: "The Throw" (Gesture.Pan() → withSpring + Haptics.impactAsync(Heavy), the LED color-snap). The theme toggle only swaps palette tokens and LCD polarity (glowing-on-black ⇄ dark-digit positive); the spring physics, gesture, and reduced-motion gate are shared. One animation codebase, two skins — exactly what "same app, two themes" means.

Parallel curriculum

One Reanimated primitive per version

Mirroring the W5D3 "Cadence" CSS progression, each version ships exactly one new motion primitive alongside its locked Expo teaching beat (feature brief §1). Pulse's signature moment ("The Throw") lands at v3 — the swipe-to-complete gesture beat — and is identical across both themes. Every animation is gated behind useReducedMotion().
